```python
def merge_sort(arr: list[int]) -> list[int]:
    """Sort a list of integers in ascending order using merge sort."""
    if len(arr) <= 1:
        return arr

    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    return merge(left, right)


def merge(left: list[int], right: list[int]) -> list[int]:
    """Merge two sorted integer lists into one sorted, stable result."""
    result = []
    i = j = 0

    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1

    result += left[i:]
    result += right[j:]

    return result
```

Bugs found and fixed:

1. **Wrong index used when taking from `right`**

   * Original:

     ```python
     result.append(right[i])
     ```
   * Problem: `i` is the index for `left`, not `right`. This can append the wrong element or cause incorrect behavior.
   * Fix:

     ```python
     result.append(right[j])
     ```

2. **Wrong slice used for leftover elements from `right`**

   * Original:

     ```python
     result += right[:j]
     ```
   * Problem: this re-adds the part of `right` that was already merged, instead of the remaining unmerged part.
   * Fix:

     ```python
     result += right[j:]
     ```

3. **Stability requirement**

   * The code already used:

     ```python
     if left[i] <= right[j]:
     ```
   * This is correct for a stable merge sort, because when values are equal, the element from `left` is taken first, preserving original relative order.
   * No change was needed here, but it is an important correctness point.
