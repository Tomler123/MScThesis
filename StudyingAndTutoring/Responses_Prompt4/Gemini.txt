Here is the complete and rigorous complexity analysis of the `mergeSort` algorithm.

### **(a) Recurrence Relation**

The recurrence relation $T(n)$ describes the total time required to sort an array of size $n$. For this algorithm, the relation is:

$$T(n) = 2T(n/2) + O(n)$$

**Justification for each term:**
* **$2T(n/2)$:** The algorithm computes `mid = n / 2` and makes two recursive calls on arrays of size $n/2$ (`left` and `right`). Each of these calls takes $T(n/2)$ time, yielding $2T(n/2)$.
* **$O(n)$:** After the recursive calls return, the `merge(left, right)` function is called. As stated in the pseudocode comments, merging two sorted halves of total size $n$ takes linear time, or $O(n)$.
* **Base Case:** When $n \le 1$, the algorithm immediately returns the array, which takes constant time, $O(1)$.

---

### **(b) Solving the Recurrence**

We can solve this recurrence relation using the **Master Theorem**. The Master Theorem provides an asymptotic bound for recurrence relations of the form:

$$T(n) = aT(n/b) + f(n)$$

where $a \ge 1$ and $b > 1$ are constants, and $f(n)$ is an asymptotically positive function. The theorem consists of three cases based on the comparison between $f(n)$ and $n^{\log_b a}$:

1.  **Case 1:** If $f(n) = O(n^{\log_b a - \epsilon})$ for some constant $\epsilon > 0$, then $T(n) = \Theta(n^{\log_b a})$.
2.  **Case 2:** If $f(n) = \Theta(n^{\log_b a})$, then $T(n) = \Theta(n^{\log_b a} \log n)$. *(Note: This extends to $f(n) = \Theta(n^{\log_b a} \log^k n) \implies T(n) = \Theta(n^{\log_b a} \log^{k+1} n)$).*
3.  **Case 3:** If $f(n) = \Omega(n^{\log_b a + \epsilon})$ for some constant $\epsilon > 0$, and if $af(n/b) \le cf(n)$ for some constant $c < 1$ and all sufficiently large $n$ (regularity condition), then $T(n) = \Theta(f(n))$.

**Applying the Master Theorem:**
From our recurrence $T(n) = 2T(n/2) + O(n)$, we identify:
* $a = 2$
* $b = 2$
* $f(n) = O(n)$

First, we compute the critical polynomial $n^{\log_b a}$:
$$n^{\log_2 2} = n^1 = n$$

Now, we compare $f(n)$ to $n^{\log_b a}$:
$$f(n) = O(n) \text{ and } n^{\log_b a} = n$$

Because $f(n) = \Theta(n^{\log_b a})$, we are exactly in **Case 2** of the Master Theorem. 

Applying Case 2, we multiply the critical polynomial by $\log n$, resulting in a closed-form Big-O bound of:
$$T(n) = O(n \log n)$$

---

### **(c) Intuitive Explanation**


The recursion tree for this algorithm has a depth of $\log_2 n$ because the array is continuously halved until it reaches a base case size of 1. At every single level of this tree, the total amount of work done by the `merge` function across all subproblems combined is exactly proportional to $n$ (e.g., one merge of size $n$, two merges of size $n/2$, four merges of size $n/4$). Since there are $\log_2 n$ levels and $O(n)$ work is performed at each level, the total time complexity is simply the product of the two dimensions. Therefore, the overall running time perfectly aligns with $O(n \log n)$.

### **Practice Exercises**

**Exercise 1:**
Find the Big-O time complexity for the recurrence relation $T(n) = 3T(n/2) + n^2$.

**Solution:**
1. Identify constants: $a = 3$, $b = 2$, $f(n) = n^2$.
2. Compute $n^{\log_b a}$: $n^{\log_2 3} \approx n^{1.58}$.
3. Compare: $f(n) = n^2$ grows polynomially faster than $n^{1.58}$. Therefore, $f(n) = \Omega(n^{\log_2 3 + \epsilon})$ for $\epsilon \approx 0.42$. 
4. This is **Case 3**. We must verify the regularity condition: $af(n/b) \le cf(n) \implies 3(n/2)^2 \le cn^2 \implies \frac{3}{4}n^2 \le cn^2$. This holds true for $c = 3/4$, which is strictly less than 1.
5. Result: **$T(n) = O(n^2)$**.

**Exercise 2:**
Find the Big-O time complexity for the recurrence relation $T(n) = 8T(n/2) + n^3$.

**Solution:**
1. Identify constants: $a = 8$, $b = 2$, $f(n) = n^3$.
2. Compute $n^{\log_b a}$: $n^{\log_2 8} = n^3$.
3. Compare: $f(n) = n^3$ and the critical polynomial is also $n^3$. Therefore, $f(n) = \Theta(n^{\log_b a})$.
4. This is **Case 2**.
5. Result: **$T(n) = O(n^3 \log n)$**.
