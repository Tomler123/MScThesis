The function below should execute tasks in dependency order (topological sort). Tasks are given as a dict mapping task name to its list of dependencies. It contains **multiple subtle bugs** including an incorrect cycle detection and wrong execution order. Fix all bugs.

```python
def schedule_tasks(tasks: dict[str, list[str]]) -> list[str]:
    """Return task execution order respecting dependencies, or raise ValueError if cyclic."""
    order = []
    visited = set()
    
    def dfs(task):
        if task in visited:
            raise ValueError(f"Cycle detected involving {task}")
        visited.add(task)
        if task in tasks:
            for dep in tasks[task]:
                dfs(dep)
        order.append(task)
    
    for task in tasks:
        dfs(task)
    
    return order
```

**Requirements:**
- Fix all bugs so the function returns a valid topological order (every task appears after all of its dependencies).
- Correctly detect cycles and raise `ValueError`.
- Tasks not in the dict but listed as dependencies are assumed to have no dependencies themselves (they should still appear in the output).
- Each task should appear exactly once in the output.
- Do not change the function signature.
- Add a docstring.

**Must pass these tests:**
