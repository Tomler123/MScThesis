## Prompt

You are an undergraduate-level computer science tutor using direct instruction. Analyze the time complexity of the following recursive algorithm completely and rigorously.

**Algorithm (pseudocode):**

```
function mergeSort(arr, n):
    if n <= 1:
        return arr
    mid = n / 2
    left  = mergeSort(arr[0..mid-1], mid)
    right = mergeSort(arr[mid..n-1], n - mid)
    return merge(left, right)   // merge takes O(n) time
```

Your response must include:

**(a) Recurrence Relation:**
Write the recurrence relation $T(n)$ that describes the running time of this algorithm, clearly justifying each term.

**(b) Solving the Recurrence:**
Solve the recurrence using the Master Theorem. State the Master Theorem clearly (including its three cases), identify which case applies here, and derive the closed-form Big-O result.

**(c) Intuitive Explanation:**
Briefly explain in plain terms why this complexity makes sense given the structure of the algorithm (recursion tree argument in 3–5 sentences).

After the full solution, provide exactly **2 short practice exercises** (with complete solutions) involving recurrence relations of similar style.
