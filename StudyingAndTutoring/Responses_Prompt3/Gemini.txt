### 1. Key Ideas

**What Big-O Notation Measures**
Big-O notation is a mathematical tool used in computer science to describe the performance or complexity of an algorithm. Specifically, it measures the worst-case growth rate of an algorithm's runtime (or space requirements) as the input size, denoted as $n$, approaches infinity. It helps us understand how an algorithm will scale when processing massive datasets.

**The Formal Definition**
Mathematically, a function $f(n) = O(g(n))$ if there exist positive constants $c$ and $n_0$ such that:
$$0 \le f(n) \le c \cdot g(n) \text{ for all } n \ge n_0$$
In simpler terms, past a certain input size ($n_0$), the growth rate of your algorithm's runtime ($f(n)$) will never exceed a scaled version of a simpler function ($g(n)$). We say $g(n)$ is the asymptotic upper bound.

**Common Complexity Classes**

Here are the most common time complexity classes, listed in increasing order of growth rate (from fastest/most efficient to slowest/least efficient):
$$O(1),\quad O(\log n),\quad O(n),\quad O(n \log n),\quad O(n^2),\quad O(2^n)$$

**Dropping Constants and Lower-Order Terms**
When analyzing algorithms, we care about the dominant trend as $n$ becomes extremely large. 
* **Lower-order terms** are dropped because they become mathematically negligible. For instance, in the expression $n^2 + n$, as $n$ reaches 1,000,000, the $n^2$ term dominates the value so thoroughly that the $+ n$ has virtually no impact on the overall growth curve.
* **Constant factors** are dropped because they are hardware and compiler dependent. An algorithm that takes $5n$ steps and one that takes $200n$ steps both scale linearly. We abstract away the constants to focus purely on the algorithmic growth rate, simplifying $O(5n^2 + 10n + 3)$ strictly to $O(n^2)$.

### 2. Worked Example

Let's analyze the time complexity of the following pseudocode:

```text
for i from 1 to n:
    for j from i to n:
        print(i, j)
```

**Step-by-step Analysis:**
1.  **Understand the loop structure:** We have two nested loops. The outer loop variable `i` goes from $1$ to $n$. The inner loop variable `j` does *not* always go from $1$ to $n$; it starts at the current value of `i`.
2.  **Count the operations:** We need to find the exact number of times the `print(i, j)` statement executes.
    * When $i = 1$, the inner loop runs from $1$ to $n$, executing $n$ times.
    * When $i = 2$, the inner loop runs from $2$ to $n$, executing $n - 1$ times.
    * When $i = 3$, the inner loop executes $n - 2$ times.
    * ...
    * When $i = n$, the inner loop runs from $n$ to $n$, executing $1$ time.
3.  **Express as a function of $n$:** The total number of `print` operations is the sum of these executions:
    $$\text{Total operations} = n + (n - 1) + (n - 2) + \dots + 2 + 1$$
4.  **Simplify mathematically:** This is a standard arithmetic series. The sum of the first $n$ integers is calculated by the formula:
    $$f(n) = \frac{n(n + 1)}{2} = \frac{1}{2}n^2 + \frac{1}{2}n$$
5.  **Convert to Big-O:** * First, drop the lower-order term ($\frac{1}{2}n$), leaving $\frac{1}{2}n^2$.
    * Second, drop the constant multiplier ($\frac{1}{2}$).
    * The resulting asymptotic complexity is $O(n^2)$.

### 3. Practice Exercises

**Exercise 1: Sequential Loops**
```text
for i from 1 to n:
    print(i)

for j from 1 to n:
    print(j)
```
**Solution 1:** These loops are unnested (sequential). 
* The first loop iterates exactly $n$ times, costing $O(n)$.
* The second loop also iterates exactly $n$ times, costing $O(n)$.
Since they happen one after the other, we add their complexities: $O(n) + O(n) = O(2n)$. Following the rule to drop constant factors, the $2$ is discarded. 
**Final Complexity:** $O(n)$

**Exercise 2: Halving the Input**
```text
k = n
while k > 1:
    print(k)
    k = k / 2
```
**Solution 2:**
In this loop, the variable `k` starts at $n$ and is divided by 2 at each step. 
* Iteration 1: $k = n$
* Iteration 2: $k = n/2$
* Iteration 3: $k = n/4$
The loop terminates when $k$ reaches 1. The total number of iterations is determined by how many times we can divide $n$ by 2 before getting 1. Mathematically, this is expressed as $\log_2 n$. Because logarithmic growth represents the number of halving steps required to reduce a value to 1, the base of the logarithm is dropped in standard Big-O notation.
**Final Complexity:** $O(\log n)$
