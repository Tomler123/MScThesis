# Complexity Analysis of Merge Sort

---

## (a) Recurrence Relation

$$T(n) = 2T\!\left(\frac{n}{2}\right) + O(n)$$

**Justification of each term:**

- **$T(1) = O(1)$** — the base case (`n ≤ 1`) does constant work and returns immediately.
- **$2T\!\left(\tfrac{n}{2}\right)$** — the algorithm makes exactly two recursive calls, each on a subarray of size $\lfloor n/2 \rfloor$ (for simplicity we treat $n/2$ as exact; the floor/ceiling distinction doesn't affect the asymptotic result).
- **$O(n)$** — the `merge` step linearly scans both halves to produce a sorted array of size $n$, taking time proportional to $n$.

Written precisely:

$$T(n) = \begin{cases} O(1) & \text{if } n \le 1 \\ 2T\!\left(\dfrac{n}{2}\right) + cn & \text{if } n > 1 \end{cases}$$

where $c > 0$ is some constant representing the per-element work done during merging.

---

## (b) Solving the Recurrence — Master Theorem

### The Master Theorem (stated fully)

Given a recurrence of the form:

$$T(n) = aT\!\left(\frac{n}{b}\right) + f(n), \quad a \ge 1,\ b > 1$$

define the **watershed function** $n^{\log_b a}$, and compare it to $f(n)$:

| Case | Condition | Result |
|------|-----------|--------|
| **Case 1** | $f(n) = O\!\left(n^{\log_b a - \varepsilon}\right)$ for some $\varepsilon > 0$ | $T(n) = \Theta\!\left(n^{\log_b a}\right)$ |
| **Case 2** | $f(n) = \Theta\!\left(n^{\log_b a}\right)$ | $T(n) = \Theta\!\left(n^{\log_b a} \cdot \log n\right)$ |
| **Case 3** | $f(n) = \Omega\!\left(n^{\log_b a + \varepsilon}\right)$ for some $\varepsilon > 0$, and $af(n/b) \le kf(n)$ for some $k < 1$ | $T(n) = \Theta(f(n))$ |

### Identifying the parameters

From $T(n) = 2T(n/2) + cn$:

$$a = 2, \quad b = 2, \quad f(n) = cn$$

Compute the watershed:

$$n^{\log_b a} = n^{\log_2 2} = n^1 = n$$

### Applying the theorem

Compare $f(n) = cn$ with the watershed $n^{\log_2 2} = n$:

$$f(n) = \Theta(n) = \Theta\!\left(n^{\log_2 2}\right)$$

This matches **Case 2** exactly ($f(n)$ grows at the same rate as the watershed).

### Conclusion

$$\boxed{T(n) = \Theta(n \log n)}$$

---

## (c) Intuitive Explanation — Recursion Tree Argument

Picture the recursion as a tree. At the **root (level 0)**, the algorithm works on an array of size $n$ and pays $cn$ for the merge. At **level 1**, there are 2 subproblems of size $n/2$, each paying $c(n/2)$, for a combined cost of $cn$. At **level 2**, there are 4 subproblems of size $n/4$, again totaling $cn$.

This pattern holds all the way down: **every level of the tree costs exactly $cn$ in merge work.** The tree has depth $\log_2 n$ (since we halve the input at each level until we reach size 1). Multiplying the per-level cost by the number of levels gives a total cost of $cn \cdot \log_2 n = O(n \log n)$.

The $\log n$ factor comes entirely from the **depth of the recursion**, while the $n$ factor comes from the **linear merge at each level** — neither dominates the other, which is why they multiply rather than one absorbing the other.

## Practice Exercises

---

### Exercise 1

**Recurrence:** $T(n) = 4T(n/2) + n$, with $T(1) = O(1)$.

**Solve for the closed-form $\Theta$ bound.**

**Solution:**

Identify parameters: $a = 4$, $b = 2$, $f(n) = n$.

Compute the watershed:
$$n^{\log_2 4} = n^2$$

Compare: $f(n) = n = O(n^{2 - 1})$, i.e., $f(n) = O(n^{\log_b a - \varepsilon})$ with $\varepsilon = 1 > 0$.

This is **Case 1** of the Master Theorem.

$$\boxed{T(n) = \Theta(n^2)}$$

**Intuition:** The recursion fans out aggressively (4 branches per level) while the merge-like work grows only linearly. The leaf-level work dominates, giving $\Theta(n^{\log_2 4}) = \Theta(n^2)$.

---

### Exercise 2

**Recurrence:** $T(n) = 2T(n/2) + n^2$, with $T(1) = O(1)$.

**Solve for the closed-form $\Theta$ bound.**

**Solution:**

Identify parameters: $a = 2$, $b = 2$, $f(n) = n^2$.

Compute the watershed:
$$n^{\log_2 2} = n^1 = n$$

Compare: $f(n) = n^2 = \Omega(n^{1+1})$, i.e., $f(n) = \Omega(n^{\log_b a + \varepsilon})$ with $\varepsilon = 1 > 0$.

Verify the regularity condition for Case 3:
$$af\!\left(\frac{n}{b}\right) = 2 \cdot \left(\frac{n}{2}\right)^2 = 2 \cdot \frac{n^2}{4} = \frac{n^2}{2} = \frac{1}{2} f(n) \le kf(n) \text{ for } k = \tfrac{1}{2} < 1 \checkmark$$

This is **Case 3** of the Master Theorem.

$$\boxed{T(n) = \Theta(n^2)}$$

**Intuition:** Here the non-recursive work $n^2$ grows so fast that the root dominates — summing across all levels of the recursion tree, the top level alone accounts for an $\Omega(n^2)$ share, so the total is $\Theta(n^2)$. The recursion itself is negligible by comparison.
