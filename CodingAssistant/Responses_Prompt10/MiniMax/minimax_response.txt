## Implementation Explanation

### 1. Section Inheritance (Including Forward References)

**How it works:**

1. When parsing a section header with inheritance syntax `[section : parent]`:
   - If the parent section **already exists** in `section_dicts`, its key-value pairs are copied into the new section immediately
   - If the parent section **doesn't exist yet** (forward reference), the inheritance request is stored in `pending_inheritances` as a tuple `(section_dict, parent_name)`

2. After all lines are processed, any remaining pending inheritances are resolved by iterating through them and copying parent values if the parent now exists

**Edge Cases Handled:**
- **Forward references**: `[production : defaults]` when `defaults` is defined later works because we resolve pending inheritances after the first pass
- **Parent never defined**: If a parent is referenced but never exists, the inheritance is silently ignored (the spec doesn't define error handling)
- **Circular inheritance**: Not explicitly handled - if `A : B` and `B : A` were defined, the second resolution would fail gracefully without infinite loops

### 2. Nested Section Name Resolution

The specification states `[parent.child]` creates nested dicts. The key insight from `config4` is:

**Resolution rule**: The path is split on the **rightmost** dot (`rsplit('.', 1)`):
- `a.b` ŌåÆ base_path = `["a"]`, final = `"b"` ŌåÆ creates `result["a"]["b"]`
- `a.b.c` ŌåÆ base_path = `["a", "b"]`, final = `"c"` ŌåÆ creates `result["a"]["b"]["c"]`

**Key placement**: Keys defined under a nested section go **into the deepest nested dict** created by the path:
```python
[a.b]
x = 1        # Goes to result["a"]["b"] = {"x": 1}

[a.b.c]
y = 2        # Goes to result["a"]["b"]["c"] = {"y": 2}
# Result: {"a": {"b": {"x": 1, "c": {"y": 2}}}}
```

### 3. Design Decisions (Spec Ambiguities)

| Ambiguity | Decision |
|----------|----------|
| **Float vs Integer** | Integer checked first (more specific), then float. So `42` ŌåÆ `int`, `42.0` ŌåÆ `float` |
| **Empty string value** | `empty =` becomes `""` (empty string), not `None` or an error |
| **Negative numbers** | Checked with `value.startswith('-') and value[1:].isdigit()` to avoid accepting `-` alone |
| **Float format** | Any value with a decimal point is tried as float (e.g., `-3.14`, `0.5`) |
| **Unresolved inheritance** | Silently ignored - no error raised (spec is silent on error handling) |
| **Section vs nested key naming** | Sections are tracked by their full dotted path for inheritance lookup, but stored by their final component name in the result dict |
| **rsplit vs split for nested paths** | Used `rsplit('.', 1)` so `a.b.c` ŌåÆ `["a.b", "c"]` rather than `["a", "b.c"]`, ensuring consistent right-associative nesting |
