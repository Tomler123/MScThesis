```python
def rle_encode(s: str) -> str:
    """Compress a string using run-length encoding.

    Each run of consecutive identical characters is replaced by
    <count><char>, where count is always present (even for single chars).

    Args:
        s: The input string to encode.

    Returns:
        The run-length encoded string, e.g. "aaabbc" -> "3a2b1c".
    """
    if not s:
        return ""
    result = []
    count = 1
    for i in range(1, len(s)):
        if s[i] == s[i - 1]:
            count += 1
        else:
            result.append(f"{count}{s[i - 1]}")
            count = 1
    result.append(f"{count}{s[-1]}")
    return "".join(result)


def rle_decode(s: str) -> str:
    """Decompress a run-length encoded string.

    Parses alternating sequences of digits (multi-digit supported) and
    a single character, expanding each pair into a repeated character run.

    Args:
        s: The run-length encoded string, e.g. "3a2b1c".

    Returns:
        The decoded original string, e.g. "aaabbc".
    """
    if not s:
        return ""
    result = []
    i = 0
    while i < len(s):
        j = i
        while j < len(s) and s[j].isdigit():
            j += 1
        count = int(s[i:j])
        result.append(s[j] * count)
        i = j + 1
    return "".join(result)
```

## How multi-digit counts are handled

In `rle_decode`, the inner `while` loop advances pointer `j` greedily as long as `s[j].isdigit()`, collecting all consecutive digit characters before the next letter. The entire digit substring `s[i:j]` is then converted to an integer in one shot via `int(...)`, which naturally handles any number of digits — so `"12a"` reads `12`, not `1` then `2`.

## Why the roundtrip property holds

`rle_encode` partitions the input into maximal runs and emits exactly `<count><char>` for each — no information is lost, and every character in the output is either a digit (part of a count) or a letter (the run character). `rle_decode` inverts this bijection precisely: it reads all leading digits as the count and the next character as the payload, then repeats it. Since the input to `rle_encode` contains only letters, digits in the encoded output can only ever be counts, so the decoder's greedy digit scan is unambiguous and the roundtrip `rle_decode(rle_encode(s)) == s` is guaranteed.
