## Fixed Code

```python
def merge_sort(arr: list[int]) -> list[int]:
    """Sort a list of integers using merge sort (stable, O(n log n)).

    Args:
        arr: The list of integers to sort.

    Returns:
        A new sorted list of integers in ascending order.
    """
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)


def merge(left: list[int], right: list[int]) -> list[int]:
    """Merge two sorted lists into one sorted list (stable).

    Args:
        left: A sorted list of integers.
        right: A sorted list of integers.

    Returns:
        A single merged sorted list.
    """
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])  # ← Bug 1 fixed
            j += 1
    result += left[i:]
    result += right[j:]              # ← Bug 2 fixed
    return result
```

## Bugs Found & Fixed

### Bug 1 — Wrong index in the `else` branch
| | Detail |
|---|---|
| **Location** | `merge()`, inside the `while` loop's `else` block |
| **Buggy line** | `result.append(right[i])` |
| **Fixed line** | `result.append(right[j])` |
| **What went wrong** | When `right[j]` is smaller, the code correctly increments `j` — but it was indexing with `i` (the *left* pointer) instead of `j`. This reads the wrong element from `right`, causes an `IndexError` whenever `i >= len(right)`, and produces an incorrect sort order otherwise. |

### Bug 2 — Wrong slice for the `right` remainder
| | Detail |
|---|---|
| **Location** | `merge()`, after the `while` loop |
| **Buggy line** | `result += right[:j]` |
| **Fixed line** | `result += right[j:]` |
| **What went wrong** | `right[:j]` appends the *already-consumed* prefix of `right` — elements that were already added during the loop — effectively duplicating them and discarding the unprocessed tail. The correct slice is `right[j:]`, which appends everything from the current position `j` to the end (the remaining, unmerged elements). The analogous line `left[i:]` on the previous line was correct all along. |

### Stability Note
The fix in Bug 1 also preserves **sort stability**. The condition `left[i] <= right[j]` (using `<=`) ensures that when two elements are equal, the one from the *left* subarray (which appeared earlier in the original list) is always placed first. If the condition were `<` instead, equal elements from the right would "jump ahead," breaking stability.
