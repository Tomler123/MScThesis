Here is the corrected implementation of the merge sort algorithm.

```python
def merge_sort(arr: list[int]) -> list[int]:
    """
    Sorts a list of integers in ascending order using the merge sort algorithm.
    
    This implementation is recursive and maintains stability (equal elements 
    retain their original relative order).
    """
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    return merge(left, right)

def merge(left: list[int], right: list[int]) -> list[int]:
    """
    Merges two sorted lists into a single sorted list.
    
    Uses two pointers to compare elements from the left and right lists,
    appending the smaller element to the result list to maintain order.
    """
    result = []
    i = j = 0
    
    # Compare elements from both lists and merge them in order
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            # Bug 1 Fixed: Changed right[i] to right[j]
            result.append(right[j])
            j += 1
            
    # Append any remaining elements from either list
    result += left[i:]
    # Bug 2 Fixed: Changed right[:j] to right[j:]
    result += right[j:]
    
    return result
```

---

### **Bugs Found and Fixed**

#### **1. Index Mismatch in `merge` loop**
* **Original Code:** `result.append(right[i])`
* **The Problem:** Inside the `else` block (when the element in the `right` list is smaller), the code was attempting to access `right[i]`. Since `i` is the pointer for the `left` list, this would either pull the wrong value or cause an `IndexError` if `i` exceeded the length of the `right` list.
* **The Fix:** Changed `right[i]` to `right[j]` to correctly use the pointer associated with the `right` list.

#### **2. Incorrect Slicing for Remaining Elements**
* **Original Code:** `result += right[:j]`
* **The Problem:** At the end of the merge process, you must append the *remaining* elements that weren't reached by the pointers. The original code used the slice `[:j]`, which takes elements from the **beginning** of the list up to the current pointer—meaning it was re-adding elements that were already in the `result` list.
* **The Fix:** Changed `right[:j]` to `right[j:]` to correctly capture only the elements from the current pointer to the end of the list.

### **Stability Check**
The sort remains **stable** because of the condition `if left[i] <= right[j]:`. By choosing the element from the `left` list when values are equal, we ensure that elements that appeared earlier in the original array stay earlier in the sorted array.
