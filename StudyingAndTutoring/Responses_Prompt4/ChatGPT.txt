# Complexity Analysis of a Recursive Algorithm

Consider the recursive algorithm:

```text
function mergeSort(arr, n):
    if n <= 1:
        return arr
    mid = n / 2
    left  = mergeSort(arr[0..mid-1], mid)
    right = mergeSort(arr[mid..n-1], n - mid)
    return merge(left, right)   // merge takes O(n) time
```

## (a) Recurrence Relation

Let (T(n)) be the running time of `mergeSort` on an input of size (n).

### Base case

If (n \le 1), the algorithm returns immediately. This takes constant time, so:

[
T(1)=\Theta(1)
]

### Recursive case

When (n>1), the algorithm does three things:

1. It splits the array into two parts.
2. It recursively sorts the left half of size (n/2).
3. It recursively sorts the right half of size (n/2) (assuming (n) is even for simplicity; this does not affect the asymptotic result).
4. It merges the two sorted halves in linear time, which costs (O(n)).

So the recurrence is:

[
T(n)=2T\left(\frac{n}{2}\right)+cn
]

for some constant (c>0), with base case

[
T(1)=d
]

for some constant (d>0).

### Justification of each term

* The term (2T(n/2)) appears because there are **two recursive calls**, each on a subproblem of size (n/2).
* The term (cn) represents the cost of the `merge(left, right)` operation, which is linear in the total number of elements being merged.
* Any additional work such as checking the base case, computing `mid`, and slicing boundaries is absorbed into the linear term or constant term.

Therefore, the recurrence relation is:

[
\boxed{T(n)=2T\left(\frac{n}{2}\right)+\Theta(n)}
]

with

[
\boxed{T(1)=\Theta(1)}
]

---

## (b) Solving the Recurrence Using the Master Theorem

## Statement of the Master Theorem

A recurrence of the form

[
T(n)=aT\left(\frac{n}{b}\right)+f(n)
]

with (a\ge 1), (b>1), and (f(n)) asymptotically positive, can be solved by comparing (f(n)) with

[
n^{\log_b a}
]

There are three standard cases:

### Case 1

If

[
f(n)=O\left(n^{\log_b a-\varepsilon}\right)
]

for some (\varepsilon>0), then

[
T(n)=\Theta\left(n^{\log_b a}\right)
]

### Case 2

If

[
f(n)=\Theta\left(n^{\log_b a}\log^k n\right)
]

for some (k\ge 0), then

[
T(n)=\Theta\left(n^{\log_b a}\log^{k+1} n\right)
]

### Case 3

If

[
f(n)=\Omega\left(n^{\log_b a+\varepsilon}\right)
]

for some (\varepsilon>0), and if the regularity condition

[
a f(n/b) \le c f(n)
]

holds for some constant (c<1) and sufficiently large (n), then

[
T(n)=\Theta(f(n))
]

---

## Apply the theorem to merge sort

We have the recurrence:

[
T(n)=2T\left(\frac{n}{2}\right)+cn
]

So:

* (a=2)
* (b=2)
* (f(n)=cn=\Theta(n))

Now compute:

[
n^{\log_b a}=n^{\log_2 2}=n^1=n
]

So:

[
n^{\log_b a}=\Theta(n)
]

and

[
f(n)=\Theta(n)
]

Thus (f(n)) matches (n^{\log_b a}) exactly.

This is **Case 2** of the Master Theorem with (k=0), because:

[
f(n)=\Theta\left(n^{\log_b a}\log^0 n\right)=\Theta(n)
]

Therefore:

[
T(n)=\Theta\left(n^{\log_b a}\log^{0+1} n\right)
]

which simplifies to:

[
T(n)=\Theta(n\log n)
]

Hence the time complexity is:

[
\boxed{T(n)=\Theta(n\log n)}
]

and therefore also:

[
\boxed{T(n)=O(n\log n)}
]

---

## (c) Intuitive Explanation

This result makes sense because at each level of recursion, the algorithm processes all (n) elements during the merge step. The array is repeatedly split in half, so the number of levels in the recursion tree is about (\log_2 n). Since each level costs about (n) total work, the full cost is (n+n+\cdots+n), repeated for about (\log n) levels. Multiplying these gives (n\log n). That is why merge sort is much faster than quadratic sorting methods on large inputs.

---

# Final Result

[
\boxed{T(n)=2T\left(\frac{n}{2}\right)+\Theta(n)}
]

[
\boxed{T(n)=\Theta(n\log n)}
]

# Practice Exercise 1

Solve the recurrence:

[
T(n)=4T\left(\frac{n}{2}\right)+n
]

with (T(1)=\Theta(1)).

## Solution

Here:

* (a=4)
* (b=2)
* (f(n)=n)

Compute:

[
n^{\log_b a}=n^{\log_2 4}=n^2
]

Compare (f(n)=n) with (n^2):

[
n = O(n^{2-\varepsilon})
]

Take (\varepsilon=1). So this is **Case 1** of the Master Theorem.

Therefore:

[
T(n)=\Theta(n^{\log_2 4})=\Theta(n^2)
]

So the answer is:

[
\boxed{T(n)=\Theta(n^2)}
]

# Practice Exercise 2

Solve the recurrence:

[
T(n)=2T\left(\frac{n}{2}\right)+n^2
]

with (T(1)=\Theta(1)).

## Solution

Here:

* (a=2)
* (b=2)
* (f(n)=n^2)

Compute:

[
n^{\log_b a}=n^{\log_2 2}=n
]

Compare (f(n)=n^2) with (n). Since

[
n^2=\Omega(n^{1+\varepsilon})
]

with (\varepsilon=1), this suggests **Case 3**.

Check the regularity condition:

[
a f(n/b)=2\left(\frac{n}{2}\right)^2=2\cdot \frac{n^2}{4}=\frac{n^2}{2}
]

So:

[
a f(n/b)=\frac{1}{2}n^2 \le c n^2
]

for (c=\tfrac12<1). The regularity condition holds.

Therefore, by Case 3:

[
T(n)=\Theta(f(n))=\Theta(n^2)
]

So the answer is:

[
\boxed{T(n)=\Theta(n^2)}
]
